Ruby

  • Ruby in 100 Minutes
    •
      General Ruby Information
      • "Programming is all about creating abstractions, and in order to create an abstraction we must be able to assign names to things. Variables are a way of creating a name for a piece of data."
      • "In English we read left-to-right, [...]. But when evaluating an assignment using the single equals (=), Ruby actually evaluates the right side first"
      • "Because Ruby’s integers are objects they have methods." This allows for us to call the .times method on the number 5, whereas with a language like JavaScript, you'd use a longer and harder-to-read for statement

      Variable Naming Principles:
      • use snake case where each word in the name is lowercase and connected by underscores (_)
      • are named after the meaning of their contents, not the type of their contents
      • aren’t abbreviated
      • Good variable names might be count, students_in_class, or first_lesson.

      Common String Methods
      • .length
      • .split
      • .sub and .gsub

      Combining Strings and Variables
      • Concatenation
      • Interpolation
      • Executing Code inside Interpolation

      Symbols
      • "If you’re new to programming, think of a symbol as a stripped down string that has barely any methods and no string interpolation."
      • "If you’re an experienced programmer, think of a symbol as a "named integer". It doesn’t matter what actual value the symbol references. All we care about is that any reference to that value within the VM will give back the same value. Symbols are thus defined in a global symbol table and their value cannot change."

Install Devise

  • Devise Gem on Github
    •  
	Add to the Gemfile:
		
	gem 'devise'
		
	
	Run the install in the terminal:
		
	bundle install
		
	
	Run the generator in the terminal:
		
	rails generate devise:install
		
	
	Add to config/environments/development.rb:
		
	config.action_mailer.default_url_options = { host: 'localhost', port: 3000 }
		
	
	Copy the Devise views to your app:
		
	rails g devise:views		
		
	
	Generate User model in command line:
		
	rails generate devise User
		
	
	Rake the database, create the migration file
		
	rake db:migrate

Dynamic Navigation Bar

  • DevFactor Twitter Devise Example
    •
  • DevFactor Twitter Dynamic Navigation
    •  
	(1) Set up two new partials in the views/layouts folder	
		
	_nav_user.html.erb
	_nav_guest.html.erb
		
	
	(2) Copy/paste the old nav-bar partial into the two new partials
	
	(3) If/then statement in the _nav.html.erb partial
		
	<% if !user_signed_in? %>
		<%= render '/layouts/nav_guest' %>
	<% else %>
		<%= render '/layouts/nav_user' %>
	<% end %>
		
	
	(4) Add devise links to the _nav_user navigation bar
		
	<%= link_to "Sign Out", destroy_user_session_path, :method => :delete %>
	<%= link_to "Settings", edit_user_registration_path %>

	You may want to wrap the embedded ruby in a span and add the drop-down item class like so:
	<span class="dropdown-item"><%= link_to "Settings", edit_user_registration_path %></span>
		
	
	(5) Examine the application controller. You may need to remove the before action filter	from the application controller.
		
	before_action :authenticate_user!
		
	
	(6) Add devise links to the _nav_guest navigation bar 	
		
	<%= link_to "Sign up", new_user_registration_path %>	
	<%= link_to "Sign in", new_user_session_path %>

Set Up First Quiz List View

	(1) Create new view file under views/welcome:	
		
	healthcast_quiz_home.html.erb
		
	
	(2) Enclose page content in two containers:
		
	<div class="container">
		<div class="quiz-home">
		
	
	(3) Display "grid" in the application.scss file, and other styling choices:
		
	.quiz-home {
		display: grid;
		grid-template-columns:90%;
		grid-template-rows:repeat(5, 80px);
		height:1000px;
		width:90%;
		margin-top:50px;
		grid-gap: 15px;
	}
		
	
	(4) Establish route to page in the config/routes.rb file:	
		
	get 'welcome/healthcast_quiz_home'
		
	
	(5) Set up link from index page to quiz list page	
		
	<span class="quiz-link"><%= link_to "Quiz List", welcome_healthcast_quiz_home_path %></span>

Prepare to Deploy to Heroku

	(1) Move gem 'sqlite3' to development block	
		
	group :development, :test do
	  # Call 'byebug' anywhere in the code to stop execution and get a debugger console
	  gem 'byebug', platforms: [:mri, :mingw, :x64_mingw]
	  # Adds support for Capybara system testing and selenium driver
	  gem 'capybara', '~> 2.13'
	  gem 'selenium-webdriver'
	  gem 'sqlite3'
	end
		
	
	(2) Add pg gem, rails 12 factor gem, and specify ruby version
		
	ruby '2.4.1'
	gem 'pg', '~> 0.18'
	gem 'rails_12factor', group: :production
		
	
	(3) Commit code to github
		
	git add .
	git commit -m "prepare for heroku deployment"
	git push
		
	
	(4) Create new heroku app
		
	heroku create
		
	
	(5) Push to heroku	
		
	git push heroku master
			
	
	(6) rake database
		
	heroku run rake db:migrate
